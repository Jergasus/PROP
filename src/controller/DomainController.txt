CLASS: DomainController
Package: controller
Layer: Controller

Purpose:
  Mediator between the presentation layer and the domain layer.
  The driver imports only this class — no model.* classes are exposed to the view.
  Loads the puzzle catalog from EXE/DriverHidato/hidatos/ at startup.
  Keeps an internal "active board" that the driver can validate or solve.

Public methods:
  - getCatalogSize(): number of puzzles in the catalog
  - getCatalogCaseName(i): name of catalog entry i
  - getCatalogCaseAdjacency(i): adjacency description of entry i
  - getCatalogCaseSolvable(i): whether entry i is expected to be solvable
  - getCatalogCaseDescription(i): free-text description of entry i
  - selectCase(i): loads a fresh copy of catalog entry i as the active board
  - getActiveBoardAsString(): text representation of the active board
  - isPartiallyValid(): true if the active board has no clue contradictions
  - isValidSolution(): true if the active board is a complete, correct solution
  - hasSolution(): tries to solve a copy of the active board (non-destructive)
  - solve(): solves the active board in-place; returns true if a solution exists
